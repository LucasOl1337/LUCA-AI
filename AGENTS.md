# LUCA-AI Agent Rules

Leia este arquivo antes de alterar o projeto. Use o metodo de `C:/Users/user/Desktop/DevKarpathy`: entender, medir baseline, mudar pequeno, verificar e documentar.

## Objetivo do projeto

LUCA-AI e uma interface Vite/React pequena para visualizar heartbeat, missoes e terminais delegados. Preserve a sensacao de painel tecnico escuro e direto.

## Regras de trabalho

1. Rode ou identifique uma baseline antes de mudar comportamento. Para este projeto, a baseline barata e `npm run build`.
2. Prefira mudancas locais em `src/main.jsx` e `src/styles.css` antes de criar novas camadas.
3. Nao adicione dependencia sem remover uma complexidade maior em troca.
4. Nao introduza backend, persistencia ou automacao real de terminal sem requisito explicito.
5. Mantenha a UI funcional em desktop e mobile.
6. Ao finalizar, informe arquivos alterados e comando de verificacao.

## Checklist de entrega

- `npm run build` passa.
- A mudanca e pequena e tem motivo claro.
- O caminho de reproducao esta no README ou na resposta final.
- Limitacoes ficam explicitas.
